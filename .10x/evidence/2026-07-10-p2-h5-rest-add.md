# P2 H5 REST add evidence

## Observation

`cdf add api.items <endpoint> --records '$.items' --cursor updated_at --cursor-param since` derives a REST source origin/path and host allowlist, performs the existing bounded sample discovery with the production HTTP transport, pins the snapshot, and writes schema-free TOML with an explicitly selected best-effort cursor. The resulting resource plans without another discovery request.

A partial flag set fails as usage before network contact or project writes. URLs containing userinfo, query material, or fragments are rejected rather than persisted.

## Procedure

- `cargo test -p cdf-cli`: 271 unit tests, doctor integration, and doc tests passed.
- `cargo clippy -p cdf-cli --all-targets -- -D warnings`: passed.

## Limits

The add surface creates public endpoints only. Secret-backed REST authentication remains expressible through declarative source configuration and the existing provider chain, but H5 does not mint secret references from CLI flags.
