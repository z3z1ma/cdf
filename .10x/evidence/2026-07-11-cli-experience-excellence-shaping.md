# CLI experience excellence shaping evidence

## What was observed

CDF has one renderer, JSON isolation, redaction, typed errors, and bounded event ingestion, but normal output is dominated by full-width rules, equal-weight panels, boxed tables, repeated history/summary facts, and incomplete terminal controls. The canonical replay recording prints an eight-event progress grid before repeating settlement facts in five panels. Help lists broad commands without descriptions.

## Procedure

Read current parser/render/output/progress source and tests, ran the existing binary's help path, inspected the recorded Chapter 23 terminal output, and compared primary official documentation for uv, Jujutsu, ripgrep, bat, and Cargo. Classified shared interaction laws separately from project-specific branding.

## What this supports

The evidence supports progressive disclosure, strict stdout/stderr roles, global terminal policy, compact activity verbs, borderless/narrow-adaptive rendering, short/long help, actionable coded errors, coalesced progress, and explicit P3 performance measurement.

## Limits

This is source/comparative shaping evidence. It does not prove user preference, accessibility, Windows behavior, or overhead targets. CX1-CX4 own those implementation and conformance claims.
