# P1 product WS2D completions, man pages, and help snapshots evidence

## What was observed

WS2D adds a clap-derived CLI artifact generator for `cdf-cli` behind the `cli-artifacts` feature. The generator writes:

- `crates/cdf-cli/generated/completions/`: bash, zsh, fish, and PowerShell completions generated by `clap_complete`.
- `crates/cdf-cli/generated/man/`: 42 man pages generated by `clap_mangen`, one for each visible command/subcommand path.
- `crates/cdf-cli/generated/help/`: 42 help snapshots rendered from the same clap command definition.

The generator is exposed as `cdf-generate-cli-artifacts` and release CI now:

- checks committed generated artifacts for freshness,
- regenerates `target/generated`,
- packages `target/generated/completions` and `target/generated/man` through the existing release artifact script.

Fast quality CI runs the feature-gated freshness test. The full-repo jscpd workflow excludes generated artifact directories so copied generated man/help/completion text does not distort source duplication metrics.

## What this supports

The evidence supports closing WS2D:

- bash, zsh, fish, and PowerShell completions are generated from the clap command definition;
- man pages are generated from the clap command definition;
- command/subcommand help snapshots exist as generated files;
- freshness check fails on drift by comparing committed generated artifacts against freshly generated output;
- release packaging consumes generated completions and man pages from `target/generated`;
- JSON envelopes, exit codes, and grammar semantics are preserved by existing full CLI tests plus focused parser/help checks.

## Limits

This evidence does not claim any WS3 human-output redesign, prose documentation quickstart work, or command grammar semantic change. Generated artifacts are committed snapshots; reviewers must inspect generated diffs semantically when clap help or command definitions change. CodeQL completed with 0 SARIF findings but retains the known Rust extractor macro-warning profile already present in this repository.
