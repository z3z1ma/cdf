# Supply-chain policy gate evidence

## What was observed

The repository now has an explicit local supply-chain policy for `cargo deny` and initialized cargo-vet metadata. The policy preserves advisory/source checks, makes license handling explicit, and turns cargo-vet from a missing-store failure into a passing check with a current-version exemption backlog.

## Procedure and results

- `cargo metadata --format-version=1 --locked --no-deps`: passed.
- `cargo deny check`: passed with `advisories ok, bans ok, licenses ok, sources ok`. The command still prints non-failing duplicate-version warnings from the current Arrow/DataFusion and platform dependency graph.
- `cargo audit`: passed after scanning 402 locked crate dependencies.
- `cargo vet --locked --output-format json --output-file /tmp/cdf-vet-policy-final.json`: passed with `conclusion: success`; 0 fully audited crates, 0 partially audited crates, and 385 crates vetted through current-version exemptions generated by `cargo vet init --locked`.

SARIF `target/quality/reports/codeql-rust-current.sarif` contains 0 results. The known Rust extractor diagnostic limits remained: 0 extraction errors and 1592 extraction warnings.

## What this supports

`deny.toml` now explicitly allows the current permissive SPDX license set needed by the locked graph while keeping no advisory ignores, checking workspace crate licenses, denying unknown registries, denying unknown Git sources, allowing only the crates.io index, and allowing no Git sources.

`supply-chain/` now exists and lets `cargo vet --locked` run as a real gate. The initial cargo-vet state is a review backlog, not an audit claim: current third-party versions are exempted under cargo-vet's default `safe-to-deploy` criteria, and future dependency graph changes must be reflected in cargo-vet metadata to keep the gate passing.

## Limits

No dependency versions were changed and no CI workflow was added. The policy does not create first-party audits or peer imports; `supply-chain/config.toml` is the durable backlog for replacing exemptions with stronger cargo-vet audits or imports over time.
