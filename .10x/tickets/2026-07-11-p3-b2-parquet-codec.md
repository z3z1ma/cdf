# P3 B2: ranged row-group-parallel Parquet codec

## Scope

Move Parquet behind the format driver, implement bounded footer/page metadata, projection/predicate pushdown, deterministic row-group units and parallel ranged decode for local/remote sources, and remove collected `FormatRead` batches.

## Acceptance criteria

- Multi-file and row-group jobs preserve file/row-group/row order and whole-file manifest completion.
- Projection/predicate fidelity and schema/physical provenance are exact and conformance-tested.
- Remote decode uses overlapping bounded ranges without full download when the server supports ranges; fallback is explicit spool.
- Parquet reaches the ratified envelope and jobs-invariance hashes match.

## Evidence expectations

Raw arrow-rs roofline, TLC/nested/wide datasets, range trace, malformed footer/page fuzzing, pushdown equivalence, memory/cancellation, and local/remote profiles.

## Explicit exclusions

No Parquet destination writer.

## Blockers

Depends on L5, FX1, segmentation, and the execution host.

- 2026-07-14: Release isolation found the first dominant codec defect. Raw arrow-rs decoded the downloaded 2,147,509,487-byte FineWeb file in 3.204 seconds (1,058,640 rows, 5,165,983,332 logical Arrow bytes); a release CDF local file-to-package-to-DuckDB run completed in 9.13 seconds, with package execution 8.723 seconds, decode 6.163 seconds, segment encode 6.943 seconds, persist/hash 1.816 seconds, and final destination receipt 0.213 seconds. The format boundary therefore needs a prepared per-file decode session that shares immutable metadata across deterministic units; a global Parquet cache or source-specific bypass would violate FX1's extension law.

## Review

- 2026-07-14 prepared-session slice — **Verdict: pass**. Adversarial inspection found no format identity branch, global metadata cache, duplicate decode surface, or unaccounted retained footer. The session validates that every execution unit belongs to the prepared plan, and Arrow IPC's retained footer stays ledger-owned until session drop. Residual risk is measured and remains owned here/G2: units execute serially inside one file, full remote spooling does not overlap decode, and FineWeb peak RSS is above 3 GiB.
- 2026-07-14 canonical-unit slice — **Verdict: pass for this milestone**. The scheduler/frontier contains no Parquet identity and the driver contains no executor/provider branch. Later errors stop admission and remain canonically ordered; stalled heads retain at most one outcome per admitted unit. Residual risk remains open under C2/B2: nested unit work does not yet share the partition admission permits, and mixed async I/O/decode still runs on the bounded I/O worker set.
- 2026-07-14 byte-envelope slice — adversarial self-review traced dictionary/data offsets, negative/corrupt metadata, zero-length columns, extent overflow/source bounds, overlapping/out-of-order units, prepared-footer reads, projections, and future page-index behavior. The first pass used Arrow's `byte_range()`, whose negative-metadata assertion would have introduced a panic; checked conversions replaced it before commit. No significant or critical finding remains for the current predicate-unsupported reader. Verdict: **pass for this prerequisite**. Residual risk is explicit: when predicate/page-index reads land, their metadata/data extents must be included or the no-lookback proof must become unavailable.

## Retrospective

- A deterministic unit abstraction is incomplete if it separates unit descriptions from shared immutable decode state. Recreating a parser builder can hide repeated metadata I/O even when the transport policy itself is correct. High-unit-count fixtures must count metadata boundary reads, not merely assert rows and schema.
- Unit parallelism generalizes only when the runtime schedules opaque prepared-session units and the format owns their physical meaning. The fastest implementation remains an architectural improvement because no row-group or Parquet branch entered source orchestration.
