# P3 C2: parallel partition/unit execution and canonical frontier

## Scope

Execute admitted partitions/format units concurrently through the streaming graph, implement bounded reorder/frontier, canonical limit/slicing/speculative discard, partition retry/reattest, file completion, and scope-safe state assembly.

## Acceptance criteria

- The focused production frontier produces jobs=1-identical package identity, lineage, statistics, and state authority under stalled-head jobs=2 execution; C4 owns the permanent randomized cross-archetype jobs 1/2/auto/N matrix after C3 joins native/FFI CPU admission.
- Reorder memory remains bounded with a stalled first partition; frontier-reserved and pre-accounted sources establish canonical-head memory priority before later admission.
- Focused row/batch/unit/partition boundary laws select the exact jobs=1 prefix at the compiled limit node and never assign discarded positions to output segments; C4 owns the exhaustive cross-archetype randomized limit matrix.
- Retried identity/schema changes fail/replan; duplicate attempts cannot cross the frontier.
- File positions advance only after every selected row-group/block/member unit completes.

## Evidence expectations

Focused scheduler/frontier laws, jobs goldens, limit/position cases, retry/identity chaos, slow-frontier memory traces, and cancellation/leak tests. The generated randomized archetype matrix and scaling profiles are an explicit handoff to C4 rather than duplicate ownership here.

## Explicit exclusions

No DataFusion/Python-specific tuning or distributed work.

## Blockers

None. The lifecycle-bearing canonical stream frontier and shared nested admission authority are implemented. The user ratified the exact retry policy, eligibility/reattest semantics, and precedence from the shaping research; `.10x/decisions/deterministic-source-retry-policy.md` records that authority.

## Retrospective

- Concurrent open establishment is not a reusable concurrency boundary for stream outcomes: once the canonical open resolves, the current consumer stops polling the open frontier and drains that stream.
- A bounded channel is only a payload bound after every producer has a scheduler-owned lifecycle and the admitted producer count itself remains live and replenished; detached per-partition prefetch scopes do not supply canonical completion ownership.
- A stalled-first gate was the right falsification test: it distinguished real outcome concurrency from fast immediate mock opens and prevented a concurrency claim based only on package identity.
- The reusable frontier belongs below source/format identity and must own the child stream lifecycle, not merely the future that opens it. One pending poll per admitted ordinal is the structural bound; channel capacity alone is insufficient.
- Nested parallelism needs one shared admission authority. Per-frontier arithmetic can be locally safe while still violating a process-wide `--jobs` promise across multiple open partitions.
- Retry must be an orchestration state machine, not a transport helper. Typed response classification belongs near the protocol, but attempts, elapsed budget, cancellation, reattestation, and canonical success admission have one scheduler owner.
- Cancellation is not an internal task failure. Representing cancellation-before-admission as a structured scope outcome prevents cleanup timing from changing the primary run error.
- Retry timer futures need abort-on-drop ownership; cancellation-aware selection alone is insufficient when the outer frontier drops the future before it receives the shared cancellation signal.
- Retry eligibility belongs to the planned unit, not only the source descriptor. A heterogeneous file source can truthfully advertise its strongest supported mechanism while weak individual objects remain unambiguously non-retryable.
- A retry contract needs an awaitable invocation-termination primitive. Documentation that producers “should already be joined” cannot prevent an adapter from reopening over live work.
- A task scope may surface one producer failure twice—once as the stream item and once as its join result. Equality with the retained primary proves joined termination; treating the second observation as cleanup failure silently disables retry.
- Exact remote revalidation must consume immutable per-partition plan identity, never rebuild the discovery/enumeration set. Otherwise an apparently harmless validation helper turns an N-file run into O(N²) control-plane work.
- Secret resolution belongs inside the source adapter's lifecycle-owned blocking invocation. A source-specific connection resolver at the adapter boundary is both simpler and safer than wrapping one source invocation inside another facade scope.
- Memory admission must use the largest retained working set that an admitted ordinal is allowed to materialize. Minimum working sets are tuning estimates, not safety ceilings.
- Checkpoint state needs EOF-bound processed-observation evidence as its only advancement authority. Segment positions describe produced bytes but cannot prove that a sliced source position was completely consumed.
- Cursor lag belongs at the observation window-closing boundary. Artifact joins must validate the same descriptor/schema while treating the recorded value as already safe.
- Opening cancellation is an owned protocol, not a dropped future. The opener receives the frontier token, retains the opening attempt, and returns only after cancel-and-join; the frontier must continue polling that future during cleanup.
- Canonical head priority is a memory-safety property: speculative polls may begin only after the head's worst-case reservation is secured.
- Pre-accounted sources prove priority by materializing the canonical head before speculative producers open; post-allocation validation alone is not admission authority.
- A source position may describe an attempted batch without describing a sliced output prefix. Partial evidence may retain the attempted value, but segment/checkpoint authority must omit it unless the position is explicitly slice-invariant.
